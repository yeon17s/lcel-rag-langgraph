## 요구사항

1. 사용자의 등급(VIP/일반)과 요청 금액을 입력받아, 이체 가능 여부와 필요한 추가 인증(MFA 등)을 안내하는 RAG 체인을 만드시오.
2. 특정 거래가 차단되었을 때(ID 9), 해당 차단의 근거가 되는 규제와 이로 인해 에이전트가 수행하게 될 다음 노드(ID 10)를 설명하는 '뱅킹 히스토리 분석기'를 구현하시오.
3. 사용자가 해외 송금이나 대출을 요청할 때, **ID 26(해외한도), ID 30(DSR), ID 39(투자적합성)**를 순차적으로 검증하고 하나라도 위반 시 거절 사유를 법적 근거와 함께 제시하는 에이전트를 구현하시오.
