## 요구사항

1. CrashLoopBackOff나 Pending 같은 K8s 상태값이 발생했을 때, 데이터셋(ID 20, 21)에서 원인과 확인해야 할 명령어를 추출해 제공하는 챗봇을 구현하시오.
2. "외부 유저가 접속했는데 502 에러가 난다"는 상황에서, **ID 10(Ingress) -> ID 7(Readiness) -> ID 8(Service)**로 이어지는 인프라 흐름을 추적하며 단계별 점검 리스트를 생성하는 RAG를 만드시오.
3. 특정 Pod 설정(YAML)을 분석하여, ID 2(비루트), ID 14(NetPol), ID 42(SecurityContext), ID 43(Docker 소켓) 규정에 위배되는 보안 취약점을 모두 찾아내고 수정 권고안을 작성하는 에이전트를 구현하시오.
